# نظام إدارة المدرسة

تطبيق ويب بـ Next.js 16.1.4 (App Router) لإدارة درجات الطلاب وحضورهم وغيابهم، خاص بثلاثة أنواع مستخدمين فقط:

- **مدرس مادة**: ينشئ ملف لكل فصل يدرّسه (الصف + الفصل)، يسجل أسماء الطلاب، ويرصد درجاتهم الأسبوعية (كراسة /10، تقييم /20، انضباط /10، واجب /10)، والامتحانات الشهرية (أكتوبر/نوفمبر من 15 درجة)، وامتحان نصف العام (من 30 درجة). مادة النشاط فقط تُرصد كدرجة واحدة من 100 آخر الترم. أى نقص عن 50% (أو 70% فى مادة الدين) يُحسب "راسب" تلقائيًا. كل مدرس يشوف فصوله فقط.
- **شؤون الطلبة**: ينشئ كشف حضور لكل فصل، يسجل أسماء الطلاب، ويرصد الحضور والغياب (بعذر / بدون عذر) أسبوعيًا، مع حساب نسبة الغياب الإجمالية تلقائيًا.
- **المدير**: حساب ثابت مُهيأ مسبقًا (لا يمر بالتسجيل)، يستعرض كل فصول المدرسة، يفتح ملف درجات أى مادة لأى فصل (بأسماء الطلاب)، يطّلع على نسبة الغياب فى أى فصل (بدون أسماء)، يقفل كشف الغياب فى نهاية كل شهر، ويسجل حضور وغياب المدرسين وشؤون الطلبة.

---

## 1) التقنيات المستخدمة

- **Next.js 16.1.4** (App Router + Server Actions)
- **React 19**
- **Tailwind CSS v4**
- **Prisma ORM + PostgreSQL** (لحفظ كل البيانات بشكل دائم)
- **jose** لتوقيع جلسات الدخول (JWT فى كوكى httpOnly)
- **bcryptjs** لتشفير كلمات المرور

---

## 2) التشغيل محليًا

### أ) قاعدة البيانات

أسهل طريقة مجانية: أنشئ قاعدة بيانات PostgreSQL على [Neon](https://neon.tech) (أو Supabase، أو Vercel Postgres). انسخ رابط الاتصال (Connection String).

### ب) إعداد متغيرات البيئة

```bash
cp .env.example .env
```

افتح `.env` وضع رابط قاعدة البيانات الحقيقى فى `DATABASE_URL`، وغيّر `AUTH_SECRET` لقيمة عشوائية طويلة (يمكنك توليدها بالأمر `openssl rand -base64 32`).

### ج) التثبيت والتشغيل

```bash
npm install
npx prisma db push      # ينشئ كل الجداول فى قاعدة البيانات
npm run seed             # ينشئ حساب المدير الافتراضى
npm run dev
```

افتح `http://localhost:3000`.

## 3) حساب المدير الافتراضى

يتم إنشاؤه تلقائيًا بالأمر `npm run seed` بالبيانات التالية (يمكن تغييرها):

```
البريد الإلكترونى: Ayman@gmail.com
كلمة المرور: Ayman123
```

لتغييرها قبل التشغيل، عدّل القيم فى `prisma/seed.mjs` أو مرّرها كمتغيرات بيئة:

```bash
ADMIN_EMAIL="admin@school.com" ADMIN_PASSWORD="StrongPass123" npm run seed
```

---

## 4) النشر على Vercel

1. ارفع المشروع على GitHub.
2. أنشئ مشروع جديد على [vercel.com](https://vercel.com) واربطه بالمستودع.
3. من إعدادات المشروع (Environment Variables) أضف:
   - `DATABASE_URL` → رابط قاعدة بيانات Postgres (Neon / Vercel Postgres / Supabase)
   - `AUTH_SECRET` → قيمة عشوائية طويلة
4. بعد أول نشر، شغّل مرة واحدة من جهازك (وملف `.env` يشير لنفس قاعدة البيانات):
   ```bash
   npx prisma db push
   npm run seed
   ```
5. الموقع هيشتغل مباشرة على رابط Vercel.

## 5) هيكل المشروع

```
src/
  app/
    page.tsx                 # الصفحة الرئيسية (تسجيل دخول / حساب جديد)
    register/                # تسجيل حساب مدرس أو شؤون طلبة
    login/                   # تسجيل الدخول
    teacher/                 # لوحة المدرس
      [classFileId]/         # ملف فصل: درجات أسبوعية، امتحانات، إجمالى
    shoon/                   # لوحة شؤون الطلبة
      [fileId]/               # كشف فصل: حضور وغياب أسبوعى
    admin/                   # لوحة المدير
      classes/[grade]/[section]/        # عرض فصل معين (درجات + غياب + قفل شهر)
      teachers/               # حضور وغياب المدرسين وشؤون الطلبة
  lib/
    actions/                 # Server Actions (تسجيل، درجات، حضور، إدارة)
    prisma.ts, auth.ts, constants.ts, grading.ts
  middleware.ts               # حماية المسارات حسب نوع المستخدم
prisma/
  schema.prisma               # مخطط قاعدة البيانات الكامل
  seed.mjs                    # إنشاء حساب المدير
```

## 6) ملاحظات مهمة

- كل مدرس يشوف فصوله هو فقط، ومفيش مدرس يقدر يشوف شغل مدرس تانى.
- نظام الدرجات: 12 تقييم أسبوعى (كراسة + تقييم + انضباط + واجب) + امتحانين شهريين + امتحان نصف العام = المجموع الكلى للمادة (باستثناء النشاط اللى بيتحسب من 100 مباشرة).
- النجاح/الرسوب يُحسب تلقائيًا: أقل من 50% رسوب فى كل المواد، وأقل من 70% رسوب فى مادة الدين تحديدًا.
- قفل كشف الغياب الشهر  يتم من حساب المدير فقط، ولا يمكن لشؤون الطلبة تعديل أسبوع بعد قفل شهره.
